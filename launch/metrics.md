# Launch Metrics

Track learning, not vanity.

## Core Metrics

Daily:

- Visitors
- Demo plan loads
- Forecast events created
- Calendar tab clicks
- Scenario tab clicks
- Stress test clicks
- Export JSON clicks
- Return visits

Qualitative:

- Confusion points
- Repeated feature requests
- Trust concerns
- Words users use to describe the problem
- Communities where the pitch resonated

## Success Criteria

Week 1:

- 100 unique visitors
- 10 pieces of real feedback
- 3 repeated pain points identified

Week 2:

- 500 unique visitors
- 50 demo plan loads
- 20 users creating at least one event
- 1 creator or newsletter reply

Month 1:

- 1,000 unique visitors
- 100 event-creating users
- 25 users returning
- Clear top 3 improvements to build next

## Learning Questions

- Do users understand "cash runway"?
- Do users prefer "future balance calendar" wording?
- Is the event form too much friction?
- Is demo data enough to understand the product quickly?
- Do users trust local-only browser storage?
- Which view makes the product click: Daily, Calendar, or Scenarios?
- What export/share format would people actually use?

## Analytics Note

If analytics are added later, use privacy-friendly analytics first. Do not add invasive tracking to a financial planning product.
